# 相機維修店庫存系統

管理維修單的本地系統，在瀏覽器操作，資料存在電腦裡。

## 啟動

在終端機執行：

```bash
./scripts/server.sh
```

瀏覽器打開 **http://localhost:8787** 即可使用。

## 使用方式

### 看板（首頁）

以看板方式顯示所有維修單的狀態：未開始、進行中、測試中、完成・待取件。

- 點擊卡片可進入編輯頁面
- 冰箱區預設收合，點擊可展開
- 卡片上會顯示收件天數

### 維修單（建立 / 編輯）

**建立新單：** 從導覽列點「維修單」，填寫類別、品牌、型號、序號、客戶資訊、維修描述，送出即建立。

**編輯舊單：** 在看板點擊卡片，或在維修單頁面上方搜尋欄輸入 ID、型號或客戶姓名找到維修單。可修改狀態、客戶資訊、維修描述、費用等。

**客戶自動填入：** 輸入客戶姓名時，會自動帶出過去的聯絡方式。

**維修記錄資料夾：** 編輯模式下有「開啟維修記錄資料夾」按鈕，會在 Finder 打開該維修單的 `logs/` 資料夾，可直接拖入照片、收據等檔案。

## 維修單狀態流程

```
未開始 → 進行中 → 測試中 → 完成・待取件 → 已交付
```

任何狀態都可以移至「冰箱」（暫停），之後再恢復。

## 資料位置

所有資料存在 `data/` 資料夾：

- `data/repairs/` — 每個維修單一個資料夾，內含 `item.md`（維修單資料）和 `logs/`（附件）
- `data/owners.json` — 客戶清單（系統自動維護）

## 系統需求

- macOS
- Python 3（macOS 內建）

## 注意事項

- `web/dashboard.html` 是自動產生的，請勿手動編輯
